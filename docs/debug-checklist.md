# ドギド デバッグチェックリスト

この文書は、ドギドの実機デバッグで確認した項目と、これから確認する項目をまとめた living checklist です。  
実際に違和感が解消したら `- [ ]` を `- [x]` に変えて使います。

## 現在の前提

- テスト環境ではクリエイティブモードの異常ケースも積極的に使う
- 実プレイでは `状態遷移`, `割り込み`, `クールダウン`, `暗所`, `音`, `複数敵優先` を重視して見る
- 2026-05-28 時点で、極端な小島テスト
  - `スケルトン5`
  - `スパイダー1`
  - `クリーパー1`
  で、優先順位まわりはかなり炙れている

## これまでに確認・修正した項目

### 音声キュー・割り込み

- [x] 悲鳴と通常セリフが同時再生されないようにした
- [x] 通常の敵コールは、基本的に後続メッセージで簡単に潰れないようにした
- [x] `志村！うしろ！うしろ〜！` の悲鳴が途中で通常コールに切られないようにした
- [x] `志村！うしろ！うしろ〜！` は30秒クールダウンにした
- [x] `こわかった…` 系の短文が即座に割り込まれにくいよう保護した

### 方向・視認・音認識

- [x] 視覚方向の `左 / 右` 反転を修正した
- [x] 音由来方向を、疑似判定から実サウンド packet ベースに変更した
- [x] 音方向の左右再計算を送信直前に行うようにした
- [x] 見えている敵に対して `声がする` を出しすぎないよう抑制した
- [x] 同じ敵の visual / audio を cross-check して、同一個体への重複反応を減らした
- [x] 視認チラつき対策として visible hysteresis を入れた
- [x] line-of-sight を複数点判定にして、`spider / creeper` の拾い漏れを減らした
- [x] ほぼ正面なのに `右 / 左` になりやすい問題を緩和し、正面判定幅を広げた

### 暗所・洞窟・水中

- [x] 夜の屋外を閉所暗所と誤判定しないよう調整した
- [x] 水中を閉所暗所扱いしないようにした
- [x] 水中では最初の1回だけ `……暗いのは、にがてなんやけど……。` を言い、その後は黙るようにした
- [x] 洞窟突入を `いくん？ -> お母さん許して〜！ -> ハァハァ` の段階制にした
- [x] `ハァハァ` は明るさ回復や退出で停止するようにした
- [x] `ハァハァ` の開始タイミングと停止タイミングを段階的に調整した
- [x] `心臓に悪いわ、マジで心臓に悪いわ……。` に置き換えた

### 日中燃焼・天候

- [x] 日中に水へ入って燃え残った敵への専用コメントを入れた
- [x] 対象を `skeleton` だけでなく日光燃焼系へ広げた
- [x] 朝に雨が降って燃えなくなった時の専用コメントを入れた
- [x] `燃えとらん` 系は最優先にし、その後に体数情報へ続ける形にした
- [x] `燃えとらん` 系は120秒クールダウンにした

### 複数敵・優先順位・クールダウン

- [x] 同じ敵種ではなく個体 `UUID` ベースで追跡するように整理した
- [x] 同じ unseen enemy の音に対する `1回目 / 4回目 / 10回目` ルールを入れた
- [x] 周囲4体以上では個別連打せず、`囲まれとるで！` 優先にした
- [x] `囲まれとるで！` は30秒クールダウンにした
- [x] 優先順を `倒さんの？ > 囲まれとるで > X体おる > 単体コール` に整理した
- [x] 同一構成が少し動いただけで `X体おる` を連打しないようにした
- [x] 単体コメント中に敵が増えた時だけ、即時で `XXが増えたで！` に切り替えるようにした
- [x] 既知の敵がいても、新しく至近距離に敵が来たら新規敵を優先するようにした
- [x] 真後ろ3マス以内の近接敵には `ぎゃー！ -> 志村！うしろ！うしろ〜！` を入れた

### LLM・ペルソナ・呼び名

- [x] `怖がりな関西人のおじさん` の共通 persona を prompt に明示した
- [x] `関西弁は語尾中心、単語は標準語寄り` を共通ルールにした
- [x] 日本語として崩れすぎる出力を reject して fallback に落とすようにした
- [x] `call_name` をイベント metadata と session の両方から扱えるようにした
- [x] fallback 文でもプレイヤーの呼び名を差し込めるようにした
- [x] LLM leaf が accepted か fallback かを server ログへ出すようにした
- [x] `light_crafted` prompt に残っていた完成文の例文を外した

## これから確認する項目

### LLM 実運用

- [ ] `mlx-community/Qwen3.6-35B-A3B-4bit-DWQ` で `llm_leaf ... result=accepted` が安定して出る
- [ ] `call_name` が LLM 生成文の中でも自然に使われる
- [ ] 呼び名を使いすぎないよう、会話ログ上で `5 ターンに 1 回程度` の頻度制御を入れる
- [ ] 将来ログ保存を入れたら、`dark_push_after_breath` がモンスター警告で潰れた時に `目の前にモンスターがいて割り込み` と残せるようにする
- [ ] fallback ではなく LLM 生成に切り替わった時、文体が例文の丸読みにならない
- [ ] 35B と 14B のどちらを既定にするか判断する

### 実プレイ寄りの状態遷移

- [ ] サバイバルの通常プレイで、同じ敵集団に対するコメント間隔が自然か確認する
- [ ] `combat_ended -> 安全地帯 -> aftermath` の流れが実プレイでも自然か確認する
- [ ] 洞窟入口の出入りを何度も繰り返した時に、`いくん？` と `ハァハァ` の再発が自然か確認する
- [ ] `audio only -> visual` 昇格時のコメントが重複しないか確認する

### 観測精度

- [ ] 広い平原で、ユーザーが肉眼で見えている敵を adapter も十分拾えているか確認する
- [ ] `spider` のような低い敵を、段差や草越しでも十分に拾えているか確認する
- [ ] 遠隔敵と近接敵が混在する時、優先順位がプレイ感に合っているか確認する
- [ ] 水辺で `入水 -> 燃えない -> 上陸 -> 燃える` の往復が安定して取れているか確認する

### 水辺・粉雪例外

- [ ] 浅瀬スケルトン系が見えた時、`背後クリーパー` を除き他コメントより優先される
- [ ] プレイヤーが `bow / crossbow` を使える時、浅瀬遠隔敵に `近づくな / 遠くから撃て` 分岐が出る
- [ ] プレイヤーが遠隔武器なしで、浅瀬遠隔敵が `8 ブロック以内` の時に即時で不利警告が出る
- [ ] 完全水没した遠隔敵は `いることだけ覚えとこか` 系で 1 回だけ言い、状態不変なら `5 分` 黙る
- [ ] 近接敵の `燃える -> 入水 -> 再燃焼` が 2 周以上続いた時、`忙しいやっちゃな` に集約され、その後 `5 分` 抑制される
- [ ] 粉雪に埋もれた敵がサーチ対象外になり、reveal 時だけ `ぎゃー！` が出る
