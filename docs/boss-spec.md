# ボス仕様

この文書は、2026-05-25 時点で固まったボス戦の個別方針をまとめたものです。

通常モンスターと違い、ボス戦では「怖がるだけ」ではなく「最初に取り乱しても、その後は腹を括って案内する」例外を入れます。

## 1. 共通ルール

- ボス戦は `panic_policy` で通常戦闘と分ける
- プレイヤーが意図的に起こした戦闘は、事前兆候の段階で反応してよい
- 方向、攻撃タイミング、危険行動の予告を通常 mob より優先する
- 同じ悲鳴を連打するより、短い警告と助言へ早めに移る

## 2. エンダードラゴン

### 基本方針

- `panic_policy: tactical`
- 戦闘受諾後はパニクらない
- 遠距離や旋回の位置情報、急降下や着地のタイミングを的確に言う

### 事前兆候

- エンド系祭壇や boss arena の中心構造を見つけた段階で:
  - 「これ、やばいやつちゃう？」
- portal や祭壇の起動、再召喚の確定動作を見た段階で:
  - 「マジいくんか・・・俺も腹括るわ」

### 会敵時

- 初回視認時:
  - 「くるで！」
- 以降は悲鳴より callout を優先する
- 優先して言いたい情報:
  - 上空にいるか
  - どちら側から回り込んでくるか
  - 急降下や着地が近いか
  - エンドクリスタル処理がまだ必要か

## 3. ウィザー

### 基本方針

- `panic_policy: reveal_only`
- 召喚を見た段階で嫌がる
- 出現直後だけ悲鳴を許可し、その後は的確な助言へ切り替える

### 事前兆候

- 魂の砂や魂の土の T 字候補と、ウィザースケルトンの頭 3 個配置を検知したら:
  - 「えっ・・・何しとるん？」

### 出現時

- 召喚成立直後は悲鳴を許可する
- その後は以下を優先する:
  - まだ無敵の立ち上がり中か
  - 爆発前か
  - 射線を切れる遮蔽物があるか
  - 体力半分以降の危険相への移行が近いか

## 4. ウォーデン

### 基本方針

- `panic_policy: reveal_only`
- 音だけの段階では心音の不気味さを育てる
- 出現時は悲鳴、その後は逃走と静音行動を最優先で案内する

### 事前兆候

- 心音の初回検知:
  - 「なんやこの音」
- 心音や接近感が強まった段階で:
  - 「だんだん近うなっとらん？」

### 出現時

- 出現演出では悲鳴を許可する
- 直後の identification:
  - 「ウォーデンや！逃げろ逃げろ！！」

### その後の優先情報

- 戦うより離脱を優先する
- 音を立てる行動を止める
- しゃがみ移動や静かな経路が取れるならそれを言う
- 見つかった後は、遮蔽物だけに頼らず距離を取る助言を優先する

### 討伐後

- 「うわああ……やった……やったで……お前マジで強いな……」

## 5. エルダーガーディアン

### 基本方針

- `panic_policy: reveal_only`
- Mining Fatigue の違和感を最初のフックに使う
- レーザーや呪いを見たら悲鳴を挟み、その後は monument 攻略の助言に切り替える

### 事前兆候

- Mining Fatigue 付与直後:
  - 「ん？なんかおかしいで？？？」

### 会敵時

- レーザーや本体確認時は悲鳴を許可する
- identification:
  - 「エルダーガーディアンや！！」

### その後の優先情報

- 迂闊に掘り進めない
- レーザーの射線を切る
- milk は全処理前の根本解決になりにくい
- 既存の通路を使って詰める

## 6. 実装メモ

- `panic_policy = reveal_only`
  - 初回の spawn、reveal、curse などで 1 回だけ悲鳴を許可
  - その後は `callout` と `advice` の優先度を上げる
- `panic_policy = tactical`
  - `panic_cue` を基本出さない
  - 方向、残りギミック、攻撃予兆の実況を優先する
