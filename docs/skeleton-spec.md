# スケルトン仕様

この文書は、2026-05-25 時点で固まったスケルトン個別仕様をまとめたものです。

正本の意図は「スケルトンの突然性を殺さず、たまにだけ察知できるようにする」です。

## 1. 役割

- スケルトンは `遠距離圧` と `突然被弾` を担当する敵
- 暗闇や遮蔽物の隙間から急に矢を通してくる理不尽さを、ある程度そのまま残す
- ただし完全な初見殺しにはせず、感のいいユーザーだけがわずかに予感できる余地を作る

## 2. 設計原則

- 音検知は完璧にしない
- 突然被弾して panic に入る面白さを最優先する
- 音で事前に察知できるのは一部だけに留める
- 事前警告が出ても、位置や敵種の断定は弱める

## 3. 検知ルール

### 足音と骨のガラガラ音

- 対象:
  - `minecraft:entity.skeleton.step`
  - `minecraft:entity.skeleton.ambient`
- 扱い:
  - `certainty: low`
  - 音量が小さい場合は無視してよい
  - 検知しても「なんかおるかも」程度に留める
- 禁止:
  - 未視認状態で「スケルトンがいる」と断定しない
  - 正確な位置特定をしない

### 弓を構える音または射撃準備を示す音

- 対象:
  - `minecraft:entity.skeleton.shoot`
  - 将来アダプタ側で `bow_ready` 相当が取れるならそれも含む
- 扱い:
  - 事前警告の主対象
  - 毎回は反応しない
  - 警告発生率は `30%` から `40%` を目安にする
  - 未視認なら `certainty: low` から `medium`
- 期待する反応:
  - 「ん？今なんか鳴ったな」
  - 「弓っぽい音、せえへん？」

### 矢の飛翔音

- 基本方針:
  - 原則として検知しない
  - ここで事前回避できるようにしすぎると突然感が死ぬ
- 例外:
  - かすめた近接弾を `near_miss` として取れる場合だけ、驚き演出に使ってよい
  - ただし panic の代わりにはしない

### 被弾

- 対象:
  - `player_damaged`
  - ダメージ原因が `arrow`、`projectile`、`skeleton` 系
- 扱い:
  - 最優先イベント
  - 即座に `panic` へ遷移する
  - 他の通常発話や低確度の警戒は割り込んで止める
- 期待する反応:
  - 絶叫 cue を優先する
  - セリフは「うわああっ！」「撃たれた！」のような短いものを優先する

## 4. 状態遷移方針

### `normal -> alert`

- 足音や ambient では、原則として弱い警戒に留める
- 弓系の音を拾った時だけ、確率で `alert` 相当の短い警告を出してよい
- ただし `alert` に入っても、敵位置の断定はしない

### `normal -> panic`

- 未視認でも、矢による被弾を受けたら即 `panic`
- これがスケルトン戦の面白さの核

### `alert -> panic`

- 事前に薄く察知していた場合でも、被弾した瞬間は即 `panic`
- 事前警告を出していても、被弾時の絶叫は弱めない

## 5. 発話ルール

### 未視認の音反応

- 名前を断定しない
- 方角は粗くてもよいが、出しすぎない
- 文面は短く、半信半疑のトーンにする

### 視認済みの反応

- 視認できている時は `スケルトン` を名指ししてよい
- ただし、見えているだけでは panic に寄せすぎず、「矢くるで」程度の短警告に留める

### 被弾時の反応

- 最優先は悲鳴
- 説明より先に驚きを出す
- 状況説明は aftermath で補えばよい

## 6. チューニング目安

- 足音・ambient:
  - 小音量時は無視
  - 検知しても `certainty: low`
- 弓系の事前警告:
  - 発火率 `0.3` から `0.4`
- 矢飛来:
  - 基本無反応
- 被弾:
  - 即 `panic`
  - 即絶叫 cue

## 7. ひとことで言うと

スケルトンは「見つける敵」ではなく、「たまに予感だけする敵」として扱う。
