## 1. いまどこにいるか

軽い気持ちで始めた相棒でも、層としてはすでに **ゲーム内エージェント** に近い。

| 層 | 内容 | おおよそ |
|---|---|---|
| 知覚・文脈 | biome / 持ちもの / 視認 / 音 / 取り合わせ | ある |
| 判断 | 状態機械・いつ詠む・黙る・バトル口調 | ある |
| 生成 | 川柳・雑談・各種 leaf → LLM | ある |
| 対話 | 短履歴 + 句の pin（workshop） | ある |
| 薄い学習 | soft lesson / praise・明示緩め / TTL | ある |
| 記憶 | entries / critiques / revisions / 読み訂正 | ある |

**AI を一から訓練しているわけではない。**  
世界を見て → 決め → 喋り → 反応を薄く残す、まで組んである、という意味での「AI を作っている」。

```text
[強い] 状況に反応する・喋る・詠む・戦う口調
[できた] 句を一緒に直す・soft に覚える・雑談の過適合を抑える
[弱い] 世界の取りこぼし / 句の当たり外れ / 「一緒に冒険してる感」の連続
[まだ先] 建造物・景色をちゃんと見る / 深い長期人格
```

骨格はある。完成度の残りはだいたい:

1. **よく見てる感**（観測の解像度）  
2. **外したあとの立ち直り**（関係を壊さない）  
3. **続きもの感**（薄い連続）

## 2. 「完成」の段階（定義）

機能数ではなく、プレイヤー体感で 3 段階。

| 段階 | 体感 | いま |
|---|---|---|
| **A 相棒β** | 危ないとき・面白いとき・句のときに反応。大外しが減る | 後半〜達成寄り |
| **B 一緒に遊んでる** | 同じ景色を共有。句を一緒に直せる。覚えすぎない | **入口**（ここを固める） |
| **C そのワールドの相棒** | 思い出・建造物・長い旅の話ができる | 先 |

次の完成度の主戦場は **B を固める**。  
手段の優先は **観測 → ログ改善 → 雑談の空気**。機能の山積みではない。

## 3. 完成度が上がる順（おすすめ）

### 3.1 観測の穴を埋める（いちばん効く）

プレイヤーが「見てるだろ」と思うものを materials に載せないと、LLM を賢くしても空振りする。

例:

- 近くの **水・湖・川**（「うみ」問題の本体になりやすい）  
- 旗・地下・LOS・音の正体（既存計画どおり）  
- structure / biome の `note` を haiku・chat に薄く載せる  

**完成度の定義:** 「いまここ」をプレイヤーと同じ解像度で共有している。

「うみ」を固定禁止する、は **やらない**（湖の圧縮・隣バイオームなど、プレイヤー視点では自然なことがある）。  
場の違和感は **プレイヤーが言ったとき** workshop で扱う。固定 drift リスト（旧 H6）は撤回済み。

### 3.2 実プレイ・ログ駆動の改善ループ

機能追加より、外した発話を集めて分類する方が完成に近い。

| 分類 | 打ち手 |
|---|---|
| 材料に無いのに語った | 候補・scene の取りこぼし |
| 材料はあるのに句が壊れた | 生成温度・repair・usable（音数・造語・道具 hard） |
| 言いすぎ / 被る | クールダウン・モード |
| 無反応 | 観測 or policy |

workshop の critique JSONL は、そのまま改善キューになる（プレイヤーが教師）。

### 3.3 川柳は「当たる率」より「外したあとの関係」

H1〜H5.2 で **関係性の側はかなりできた**（pin / 講評 / soft lesson / 緩め / TTL）。

足すなら（任意）:

- 直し案 1 本（材料語の並べ替えだけでも可）  
- 「前よりマシ？」の短い自己参照（lessons が効いた感）  

**やらない方がいい:**

- 固定禁止語で締め直す  
- revision の few-shot 山盛り  
- strength の複雑な段階（TTL + 明示緩めで足りる）

### 3.4 雑談の「空気」

P1〜P4 で過適合は改善済み。残り（任意）:

- none のときの自然な相槌（P5）  
- 同じ話題の連打を避ける  
- 冒険の小さな連続を **1 文だけ**（さっきの洞窟、さっきの句）

### 3.5 人格の連続（薄い長期記憶で十分）

フル人格モデルは不要。効くのは:

- 呼び方・嗜好  
- 「このワールドで一緒に見たこと」をたまに思い出す（haiku recall の一般化）  
- battle / peace のトーン切り替えのキレ  

### 3.6 大きな飛び道具は後回し

| 後でいい | 理由 |
|---|---|
| VLM 常時 | コスト大。イベント駆動の「あれ見て」だけで十分 |
| Vector RAG | 直引きと被りやすい。材料が薄い問題を先に |
| H7 LLM workshop 分類 | ルールで足りている間はノイズ |
| 巨大パッケージ整理（Phase E） | 完成度というより開発者快適さ |

## 4. 本丸（一文）

完成度を上げる本丸は **もっと賢いモデル** ではなく:

> **プレイヤーが見てる世界を、ちゃんと materials に載せ続けること**  
> **外したときは、一緒に直せる関係を壊さないこと**

soft workshop 方針は後半をすでに押さえている。  
前半（観測の解像度）をゆっくり上げるのが、「完成した相棒」にいちばん近い。

---

## 5. 気楽な進め方

締切なし前提。

1. **気が向いたセッションで実プレイ** → 気になった発話を数個メモ  
2. 「観測 / 生成 / ポリシー / 無反応」に分ける  
3. 観測ギャップなら候補を足す。生成ならプロンプト 1 行。関係なら workshop のまま  
4. 機能 PR は気が向いたら 1 本（直し案・P5・水辺候補など）

**やらないこと（このガイドの精神）:**

- 機能チェックリスト消化競争  
- プレイヤーなしの完全自動名句を目標にする  
- VLM / RAG を「完成の必須条件」にしない  
- soft lesson を hard 禁止に戻す

## 7. 候補バックログ（優先度の目安）

気が向いたときのメニュー。必須ではない。

| 優先感 | 候補 | ねらい |
|---|---|---|
| 高 | 水辺・湖・川など近傍の materials 載せ | 共有解像度・「うみ」誤解の根 |
| 高 | 実プレイログ分類 → 小さな修正 | 当たり外れの実害を減らす |
| 中 | 既存観測ギャップ（旗・地下・音） | chat の「見てる感」 |
| 中 | 直し案 1 本 / P5 fallback | 体験の角を丸く |
| 低 | haiku recall の一般化・薄い連続 | B→C 寄りの続きもの感 |
| 後 | VLM イベント駆動 / RAG / H7 / 整理 PR | 飛び道具・掃除 |

更新したらこの節と §6 だけ直せばよい。
