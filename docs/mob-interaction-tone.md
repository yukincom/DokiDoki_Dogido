# モブ反応トーン方針（公式 Minecraft に合わせる）

**日付:** 2026-07-16  
**状態:** 方針（実装前）  
**対象:** 主に `ambient` leaf（平和時の周辺モブ一言）。雑談・警告 leaf にも同じ倫理を波及させる。  
**関連:** [dialogue-design.md](dialogue-design.md) · [companion-maturity.md](companion-maturity.md) · [player-chat-casual-plan.md](player-chat-casual-plan.md)

---

## 1. 背景

実ログ例:

> ambient: 「左前のかわいい羊さん、のそのそ歩いてるやろ、**触らんとこな**」

友好モブ（羊）に対する **操作禁止** は体験としてまずい。羊毛・繁殖など、触る・関わるのがゲームの本筋だからである。

原因の一端は ambient プロンプトが中立向け正例として  
「触らんほうがええ」「近づきすぎんほうがええ」を示し、friendly でも注意口調（variation）を要求していたことにある。  
詳細な原因分析は実装 PR 時にコードへ反映する。本ドキュメントは **トーンとポリシーの正** を固定する。

## 2. 公式が示しているトーン

ロード画面 Tips（主に Bedrock の Loading tips）および公式モブ解説の骨格:

| 系統 | 公式寄りのメッセージ | 含意 |
|---|---|---|
| 動物全般 | **Be nice to animals!** | 優しく・親切に |
| イルカ | Don't kill dolphins, you monster! | 殺すな（倫理・愛嬌）。関与そのものの全面禁止ではない |
| オオカミ | 攻撃すると近くの狼も敵対する | **攻撃＝怒らせる** と説明 |
| ゾンビピグリン | 怒らせない限り攻撃してこない | **非挑発なら穏やか** |
| ハチ | 焚き火下で蜜を取る、など | 怒らせない **やり方** で関われる |
| 友好寄り Tips | 小麦でついてくる／手懐け／革を取る 等 | **関わってよい・活用してよい** |

公式の三分類（要約）:

- **パッシブ:** 争わない。攻撃しても反撃しない。飼育・繁殖・食料などに使える  
- **中立:** 普段は穏やか。攻撃・視線・条件で敵対に切り替わる  
- **敵対:** 危険（本方針の ambient 主対象外。警告 leaf 側）

公式は「触るな・近づくな」一択ではない。  
**優しく / 殺すな / 怒らせるな / やり方を工夫して関われ** が中心である。

参考:

- [Loading tips – Minecraft Wiki](https://minecraft.wiki/w/Loading_tips)  
- [Minecraft のモブについて（minecraft.net）](https://www.minecraft.net/ja-jp/article/minecraft-mobs)

## 3. ドギドの方針（要約）

1. **トーンは公式に合わせる**（Be nice / 非挑発 / 活用可）  
2. **判断はコード**（interaction stance）。LLM は言い回しのみ  
3. **友好・資源モブに操作禁止を言わない**  
4. **中立は「触るな」より「優しく・怒らせんと」**  
5. **種ごとの特殊条件**（視線・金防具・焚き火等）は stance または catalog で足す。汎用禁止語リストで全部は潰さない  

---

## 4. Interaction stance（コード用ラベル）

ambient の details に載せる想定の分類。種 ID + temperament + caution_reason から解決する。

| Stance | 典型 | ドギドが言ってよいこと | 言ってはいけないこと |
|---|---|---|---|
| **welcome_interact** | 羊・牛・豚・鶏 等 | 観察、愛嬌、毛刈り・餌・連れの示唆 | 触るな、さわるな、ほっとけ、近づくな（操作抑止） |
| **be_gentle** | 中立の多く（ラマ・ヤギ・イルカ・パンダ 等） | 優しく、怒らせんと、叩かんほうが、刺激せんと | 全面の「触るな」「関わんな」（飼育・取引を塞ぐ言い方） |
| **special_caution** | エンダーマン、条件付きピグリン 等 | 条件付き注意（目を合わせない、など） | 無関係な操作禁止の一般化 |
| **hostile** | ambient 対象外寄り | 警告 leaf へ | peace ambient で雑に「かわいい」化しない |

`temperament` 未設定時:

- カタログ上パッシブ資源種 → `welcome_interact`  
- 既知の中立種 → `be_gentle` または `special_caution`  
- 不明 → 観察のみ（`be_gentle` 寄りで安全側）

## 5. 発話トーンの正例 / 反例

### welcome_interact（羊など）

| よい | 悪い |
|---|---|
| 左前の羊さん、のそのそしてるな | かわいい羊さん、**触らんとこな** |
| 毛ふわふわやな | **ほっとけ**、**さわらんほうがええ** |
| 小麦持っとったらついてきそうやな | **近づくな** |

### be_gentle（中立）

| よい | 悪い |
|---|---|
| 優しくしとこか | 触らんほうがええ（操作禁止に聞こえる） |
| 怒らせんと大丈夫そうやな | もう敵や、やばい（敵対断定） |
| 叩かんほうがええで（刺激注意） | こわいこわい連発 |

### special_caution

| よい | 悪い |
|---|---|
| 目合わせんとこか（エンダーマン） | 全中立に同じテンプレ |
| 怒らせん方がええで（ピグリン系） | 友好羊と同じ「触るな」 |

公式の **Be nice to animals** を関西弁相棒に落とすと、中立の既定はだいたい:

- 「優しくしとこか」  
- 「怒らせんほうがええ」  
- 「刺激せんと大丈夫そうやな」

## 6. 実装への落とし込み（予定）

コード変更は別 PR。方針との対応だけ記す。

### 6.1 プロンプト（ambient）

- `neutral` 正例の『触らんほうがええ』を **やめる**  
- stance ごとに指示を分岐（上表）  
- `welcome_interact` では variation の「軽い注意から入る」を使わない  
- 中立は **Be nice / 非挑発** を明示  

### 6.2 出力ガード

- `is_style_acceptable("ambient", …)` を有効化  
- `welcome_interact` で操作禁止フレーズを reject → カタログ fallback  
- `be_gentle` では「怒らせん」「叩かん」は許容、「触らんとこ」全面禁止は拒否してよい  

### 6.3 カタログ

- `ambient_reactions.json` の neutral generic を「刺激せんかったら」等、**非挑発**寄りに揃えられるか監査  
- 種固有（ハチの焚き火、イルカの Don't kill トーン）は specific_lines で維持可  

### 6.4 やらないこと

- 羊だけのハードコード  
- ambient で一切の注意を禁止（ハチ・ピグリンが壊れる）  
- 公式Tips全文をプロンプトに貼る  
- プレイヤーへの長い攻略説明  

---

## 7. 成功条件

1. 羊など welcome 種で「触るな」系が生成・受理されない  
2. 中立は「優しく／怒らせない」トーンが主になる  
3. 公式 Tips の精神（Be nice / 条件付きの怒り）と矛盾しない  
4. 種追加は stance 表または catalog 1 行で足りる
